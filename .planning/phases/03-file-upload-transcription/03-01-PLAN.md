  <action>
Create frontend/src/hooks/useTranscribeQueue.ts implementing a sequential file processing hook:

```typescript
export interface UseTranscribeQueueReturn {
  jobs: TranscribeJob[];
  isProcessing: boolean;
  currentJobIndex: number | null;
  enqueue: (file: File) => { success: true } | { success: false; reason: string };
  removeJob: (index: number) => void;
  processQueue: () => Promise<void>;
  clearCompleted: () => void;
}

export function useTranscribeQueue(): UseTranscribeQueueReturn { ... }
```

**Internal state**: `useState<TranscribeJob[]>([])`, `useState<boolean>(false)` for isProcessing, `useState<number | null>(null)` for currentJobIndex.

**enqueue(file)**: Call `validateFile(file)`. If invalid, return `{ success: false, reason }`. If valid, create TranscribeJob: `{ file, status: 'queued', progress: 0 }`. Append to jobs array. Return `{ success: true }`. Per D-02 (multi-file).

**processQueue()**: 
- Set isProcessing=true
- Iterate jobs array from currentJobIndex (or 0) 
- For each job: update status to 'uploading', set currentJobIndex to job index
- Call `uploadAudio(job.file, (pct) => updateJobProgress(index, pct))`
- On XHR complete: update status to 'processing', progress to 100
- On response: update status to 'complete', set job.result = response
- On error: update status to 'failed', set job.error = TranscribeError
- After all jobs processed: set isProcessing=false, currentJobIndex=null
- Use AbortController for cancellation on unmount: `useEffect(() => { return () => controller.current.abort() }, [])`

**Key pattern**: Sequential means a `for...of` loop with `await` (not Promise.all). Each job waits for the previous one to complete. Per D-03.

**Progress reporting** (D-05): job.progress updates from uploadAudio callback. Status label uses job.status ('uploading' | 'processing' | 'complete' | 'failed').
