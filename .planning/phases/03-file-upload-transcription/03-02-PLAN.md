<!-- TranscribeJob from Plan 01 types — consumed by UI components -->

```typescript
interface TranscribeJob {
  file: File;
  status: 'queued' | 'uploading' | 'processing' | 'complete' | 'failed';
  progress: number; // 0-100
  result?: TranscribeResponse;
  error?: TranscribeError;
}

interface TranscribeResponse {
  text: string;
  language: string;
  usage: { prompt_tokens: number; completion_tokens: number } | null;
  processing_time_ms: number;
}

interface TranscribeError {
  error: string;
  message: string;
  code: number;
}
```

<!-- UseTranscribeQueueReturn from Plan 01 hook -->
```typescript
interface UseTranscribeQueueReturn {
  jobs: TranscribeJob[];
  isProcessing: boolean;
  currentJobIndex: number | null;
  enqueue: (file: File) => { success: true } | { success: false; reason: string };
  removeJob: (index: number) => void;
  processQueue: () => Promise<void>;
  clearCompleted: () => void;
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Spoofing | FileUploadZone file input | mitigate | Accept attribute restricts to audio extensions. Client-side validateFile() is the real gate (Plan 01). |
| T-03-07 | Tampering | ResultCard clipboard.writeText | mitigate | Browser enforces user-gesture-only clipboard access. No privilege escalation possible. |
| T-03-08 | Info Disclosure | ResultCard text display | accept | Transcription text displayed in browser memory only. Never transmitted to third party. |
| T-03-09 | DoS | FileUploadPanel enqueue flood | mitigate | Queue processes sequentially. No parallel XHR spam. 50MB per-file limit prevents individual large uploads. |
</threat_model>
