<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → Backend WS | Untrusted browser sends audio data via WebSocket to backend |
| Backend WS → vLLM WS | Backend relays to vLLM on internal network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | WS /ws/transcribe | accept | No auth needed — internal demo, same as REST /api/transcribe |
| T-04-02 | Tampering | Audio data in WS frames | mitigate | Validate frame `type` field, reject unknown types |
| T-04-03 | Repudiation | Audio stream | accept | Internal demo, no audit trail needed |
| T-04-04 | Info Disclosure | Transcription text in WS | accept | Internal network, same exposure as REST endpoint |
| T-04-05 | DoS | Connection flood | mitigate | Close vLLM connection on frontend disconnect; asyncio.cancel on cleanup |
| T-04-06 | Elevation | vLLM connection hijack | mitigate | Backend proxies WebSocket — frontend never connects directly to vLLM |
</threat_model>
