<task type="auto" tdd="false">
  <name>Task 2: Fix streaming text duplication — replace instead of append partial text</name>
  <files>frontend/src/hooks/useStreamingTranscribe.ts</files>
  <action>
    Fix the `onPartial` callback in the `StreamingClient` constructor (line 83) to replace partial text instead of appending.

    Current bug: backend `streaming.py` sends full-buffer re-transcription as `{"type": "partial", "text": "..."}` every 3 seconds. Frontend does `setPartialText((prev) => prev + text)` which accumulates: "嗯" + "嗯。嗯。" + "嗯。嗯。嗯。" = garbled output. User reports: "输出一直在重复".

    Fix: Change line 83 from:
    `(text) => setPartialText((prev) => prev + text)`
    to:
    `(text) => setPartialText(text)`

    This is correct because backend `periodic_transcription()` (line 136 of streaming.py) calls `clean_text, lang = await call_http_transcription(wav_bytes)` which re-transcribes the FULL audio buffer each cycle. The text it sends IS the complete transcription — not a delta. So replacing (not appending) is the correct behavior.

    Also update the `onFinal` callback (line 85-89) — it already does `setFinalText(text)` (replace), which is correct. No change needed there.

    After this fix, at any moment during recording, `partialText` will show the latest full transcription of all audio so far, not an ever-growing accumulation.
  </action>
  <verify>
    Start recording, speak for 10+ seconds. Verify: partial text shows coherent transcription that updates every ~3s. No duplication (e.g., NOT "嗯。嗯。嗯。"). Text replaces cleanly — each update shows current transcription, not accumulated history.
  </verify>
  <done>
    Line 83 uses `setPartialText(text)` (replace). No `prev +` append pattern exists. Speaking for 10+ seconds produces clean, non-duplicated transcription updates.
  </done>
</task>
