    - **GET** `/health`: Call vLLM `/v1/models` endpoint
    - Return `{status, latency_ms, message}`

    **backend/app/routers/transcribe.py** — D-04, D-05, D-06 audio transcription:
    - **POST** `/transcribe`: Accept `file: UploadFile`, `model: str = "Qwen3-ASR-1.7B"`
    - Read audio bytes, send to `client.audio.transcriptions.create()`
    - Return `{text, language, usage:{prompt_tokens, completion_tokens}, processing_time_ms}`
    - No audio preprocessing — forward as-is (D-05)
    - If audio fails, try chat completion with `audio_url` content type as fallback
    - Use configurable `request_timeout` from settings
